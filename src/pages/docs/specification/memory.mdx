# Memory

Memory in APC is durable context tied to a project agent.

## Location

```text
.apc/agents/<slug>/memory.md
```

## Why markdown

APC keeps memory human-readable by default:

- easy to review
- easy to diff
- easy to migrate
- not trapped in one runtime database

## Recommended shape

```markdown
# Memory — reviewer

## Identity
- Reviews behavior changes and risks

## Long-term facts
- Prefer findings-first reviews

## Recent context
- Tracking docs migration for APC
```

## Scope guidance

Good APC memory contains information that remains useful across sessions and across tools:

- stable project responsibilities
- recurring constraints
- long-lived domain facts
- durable team expectations
- decisions and their rationale

Transient scratch notes and raw session history belong in `sessions/`, not `memory.md`.

## Memory vs sessions

| | `memory.md` | `sessions/` |
|---|---|---|
| Visibility | `project` — commit if useful | `local` — gitignored by default |
| Written by | Agent (curated) | Agent (raw log) |
| Readable by | All agents, all IDEs | Local runtime only |
| Purpose | Durable facts | Full conversation history |

The intended pattern: agents run sessions locally, then extract meaningful decisions and facts into `memory.md`. The session stays local. The memory travels with the repo.

## Persistence levels

APC defines five visibility levels for `.apc/` content:

| Level | Examples | Default VCS behavior |
|---|---|---|
| `stable` | Agent definitions, skills, rules | Commit |
| `project` | `memory.md`, `AGENTS.md`, `project.json` | Commit |
| `local` | `sessions/`, `chats/`, `cache/` | Gitignored |
| `private` | Secrets, credentials, `*.secret.json` | Gitignored, never commit |
| `ephemeral` | `tmp/`, build artifacts | Gitignored |

`memory.md` is `project` visibility by default — it is safe to commit and useful for team continuity. Raw sessions are `local` and never committed unless the user explicitly curates and exports them.
