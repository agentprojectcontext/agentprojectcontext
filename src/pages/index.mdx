## Why APC

Teams increasingly maintain agent prompts, skills, rules, and memory in duplicated tool-specific
locations such as:

- `.claude/`
- `.cursor/`
- `.windsurf/`
- `.codex/`
- `.opencode/`

APC proposes a simpler model:

> A project should own one durable context layer that any compatible tool can read.

## What APC standardizes

- project metadata
- agent definitions
- reusable skills
- durable memory
- session records
- optional MCP registry hints
